# Plugin Architecture

**Version**: 0.3.0
**Last Updated**: 2026-04-30

## Overview

The Agentic SDLC VS Code Copilot plugin provides automatic trace generation and artifact validation through lifecycle hooks. This document describes the plugin architecture, file structure, and implementation details.

## Features

### 1. Trace Generation Hooks (8 Lifecycle Events)

| Hook | Trace Event | Purpose |
|------|-------------|---------|
| SessionStart | HOOK_SESSION_START | Session initialization |
| UserPromptSubmit | HOOK_USER_PROMPT | User prompt tracking |
| PreToolUse | HOOK_PRE_TOOL_USE | Tool call tracking |
| PostToolUse | HOOK_POST_TOOL_USE | Tool result tracking |
| PreCompact | HOOK_PRE_COMPACT | Context compaction |
| SubagentStart | HOOK_SUBAGENT_START | Subagent spawn |
| SubagentStop | HOOK_SUBAGENT_STOP | Subagent completion |
| Stop | HOOK_STOP | Session termination |

### 2. Artifact Validation System

Validates that agents produce required artifacts:

| Agent | Required Artifact | Validation |
|-------|-----------------|------------|
| PO | REQUIREMENTS.md | File exists, size > 0, required sections present |
| Architect | PLAN.md | File exists, size > 0, required sections present |
| CTO | PLAN.md | Approval marker present in file |
| Implementor | Code files | Minimum file count, minimum total size |
| QA Lead | QA_REPORT.md | File exists, size > 0, required sections present |
| Tech Writer | ADR.md | File exists, size > 0, required sections present |
| PR Reviewer | PR Feedback | Feedback document exists, size > 0 |

## Component Details

### 1. Plugin Manifest (`plugin.json`)

```json
{
  "version": "0.3.0",
  "hooks": "hooks.json"
}
```

- Version bumped from 0.2.0 to 0.3.0
- Added `hooks` field to register `hooks.json`

### 2. Hook Configuration (`hooks.json`)

Defines 8 lifecycle hooks with their Python commands:

```json
{
  "hooks": {
    "SessionStart": {
      "command": "python3 ${PLUGIN_ROOT}/scripts/trace_hook.py",
      "args": ["SessionStart"],
      "environment": {}
    },
    "UserPromptSubmit": { ... },
    "PreToolUse": { ... },
    "PostToolUse": { ... },
    "PreCompact": { ... },
    "SubagentStart": { ... },
    "SubagentStop": { ... },
    "Stop": { ... }
  }
}
```

All hooks use `${PLUGIN_ROOT}` token to reference scripts within the plugin directory.

### 3. Shell Wrapper (`trace_hook.sh`)

Minimal shell wrapper that delegates to Python:

```bash
#!/bin/bash
PLUGIN_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
exec python3 "$PLUGIN_ROOT/scripts/trace_hook.py" "$@"
```

- Made executable (`chmod +x`)
- Handles Python invocation
- Resolves plugin root path

### 4. Python Dispatcher (`trace_hook.py`)

Main implementation (578 lines) with two primary functions:

#### Trace Generation

- Reads hook event from command line arguments
- Extracts session ID from environment or payload
- Writes trace events to `.agentic-sdlc/state/raw/` for session binding
- Event format: `HOOK_SESSION_START`, `HOOK_USER_PROMPT`, etc.

#### Artifact Validation

- Runs on `SubagentStop` hook
- Validates agent outputs against rules in `artifact_requirements.json`
- Checks: file existence, size, sections, approval markers
- Writes validation reports to `.agentic-sdlc/state/validation_reports/`
- Logs results to TRACE.jsonl as `ARTIFACT_VALIDATION` events

#### State Management

- Creates `.agentic-sdlc/` directory structure on first run
- Auto-generates `.gitignore` entries for runtime state
- Session binding: Events in `raw/` bound to sessions in `sessions/`

### 5. Validation Configuration (`artifact_requirements.json`)

Defines validation rules for each agent:

```json
{
  "agents": {
    "po": {
      "artifacts": [
        {
          "path": "REQUIREMENTS.md",
          "required_sections": ["## User Stories", "## Acceptance Criteria"],
          "min_size_bytes": 100
        }
      ]
    }
  }
}
```

Configurable per:
- File path patterns
- Required sections (markdown headers)
- Minimum file size
- Required patterns (regex)
- Approval markers (specific text)

### 6. Standalone Validation Script (`validate_artifacts.py`)

Can be run independently for manual validation:

```bash
python3 .plugin/scripts/validate_artifacts.py <session_id>
```

- Validates all artifacts for a session
- Produces detailed report
- Exit code: 0 (all pass), 1 (failures), 2 (error)

## Event Flow

### Trace Generation (Phase 1 - Double Trigger)

1. **Hook Triggered**: VS Code Copilot lifecycle event fires
2. **Shell Wrapper**: `trace_hook.sh` receives event name
3. **Python Dispatcher**: `trace_hook.py` processes event
4. **Event Spool**: Event written to `.agentic-sdlc/state/raw/<timestamp>.jsonl`
5. **Session Binding**: Events collected and bound to session in `.agentic-sdlc/state/sessions/<session_id>/`
6. **TRACE.jsonl**: Canonical trace written to `docs/adr/XXX-feature/TRACE.jsonl`

**Note**: Phase 1 uses double-trigger - hooks write `HOOK_*` events while orchestrator continues writing canonical SDLC events (`RUN_START`, `STAGE_START`, etc.). This maintains backward compatibility during migration.

### Artifact Validation

1. **Agent Completes**: SubagentStop hook triggered
2. **Agent Identification**: Extract agent name from payload
3. **Load Rules**: Read `artifact_requirements.json` for agent
4. **Validate Artifacts**: Check file existence, size, sections, patterns
5. **Generate Report**: Write detailed report to `.agentic-sdlc/state/validation_reports/<agent>_<timestamp>.json`
6. **Log Event**: Write `ARTIFACT_VALIDATION` event to TRACE.jsonl
7. **Auto Git Ignore**: Update `.gitignore` if new files created

## Key Design Decisions

### 1. Hooks in Plugin Directory, Not Workspace

**Decision**: Hooks installed in `.plugin/` directory (plugin repository), not in `.github/hooks/` (user workspace).

**Reasons**:
- VS Code plugins are installed to `~/.config/Code/agentPlugins/...`
- Hooks run from plugin cache, not from user workspace
- No need to copy hook files to each workspace
- Single source of truth for hook configuration

### 2. Runtime State in User Workspace

**Decision**: `.agentic-sdlc/` created in user workspace (gitignored), not in plugin repository.

**Reasons**:
- Each workspace needs its own session state
- Plugin is read-only in cache, cannot write there
- State must not be committed to repo
- Auto-generated `.gitignore` prevents accidental commits

### 3. Double-Trigger Migration (Phase 1)

**Decision**: Hooks write `HOOK_*` events while orchestrator writes canonical SDLC events.

**Reasons**:
- Maintains backward compatibility with existing tools
- No changes required to orchestrator in Phase 1
- Gradual migration path to hooks-only model
- Both event types coexist safely in TRACE.jsonl

### 4. Artifact Validation on SubagentStop

**Decision**: Validation runs when subagent completes (SubagentStop hook), not on Stop hook.

**Reasons**:
- Immediate feedback on agent output
- Validation reports available for human review
- Can trigger remediation before next stage
- Logs validation event to TRACE for observability

### 5. Python Implementation

**Decision**: Main hook dispatcher written in Python, not bash or other language.

**Reasons**:
- Rich JSON processing capabilities
- Cross-platform compatibility
- Easy to extend with validation logic
- Mature ecosystem (pytest, etc. for testing)

## Extension Points

### Adding New Hooks

1. Add hook to `.plugin/hooks.json`
2. Update `trace_hook.py` to handle new event type
3. Define event payload structure
4. Add validation if needed

### Adding New Agent Validations

1. Add agent to `.plugin/config/artifact_requirements.json`
2. Define artifacts and validation rules
3. Test with `validate_artifacts.py`
4. Deploy with next plugin version

### Custom Validation Rules

Extend `validate_artifacts.py` with custom validators:

- Pattern matching (regex)
- Content validation (semantic checks)
- Cross-file validation (dependencies)
- External API calls (lint, format check)
